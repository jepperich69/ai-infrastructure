# /respond — Reviewer response loop

Runs autonomously. The user will review the output after completion. Do not pause for approval except where explicitly instructed below.

> Canonical source: `AI_auto/prompts/respond.md` — keep in sync with that file.

## Argument parsing

`$ARGUMENTS` is one of:
- `R1` — round 1, clean start
- `R2` — round 2, clean start
- `R1 --draft response_R1.tex` — round 1, continue from partial draft
- `R2 --draft response_R2.tex` — round 2, continue from partial draft

Parse the round label (R1, R2, R3...) and the optional `--draft <file>` argument.

## File conventions

All files live in `Overleaf_source/` of the current project root.

| File | Role |
|---|---|
| `reviewers_R[n].txt` | Reviewer comments — input, you read this |
| `response_R[n].tex` | Response letter — output, you write this |
| `review_log_R[n].md` | Tracking table — output, you write this |
| `main.tex` | Working manuscript — you edit this |
| `manus_R[n]_diff.tex` | Tracked-changes diff — generated at the end |

Snapshot tag: `snapshot-pre-R[n]`

## Phase 1 — Parse and structure

Parse `reviewers_R[n].txt` into a numbered comment list. Reviewer boundaries are typically marked by "Reviewer 1", "Reviewer 2" etc. Number every comment: R1.1, R1.2 ... R2.1, R2.2 ...

For each comment record:
- **ID:** R[reviewer].[comment] (e.g. R1.3)
- **Text:** verbatim quote
- **Type:** Major / Minor / Editorial
- **Requires manuscript change:** Yes / No / Maybe
- **Status:** Unaddressed (initial state)

Then identify **cross-cutting comments** — where two or more reviewers raise the same issue. Flag these explicitly:
- Cross-cutting: R1.2 + R2.5 (both ask for more discussion of limitations)

If `--draft` is given: read the draft, match each response already written to its comment ID, mark those comments as **Already addressed — carry forward**. Do not rewrite them unless they need adjustment for consistency.

---

## Referencing convention — follow this throughout all passes

Reference manuscript locations by **title only**, never by number or line:

- **Correct:** *"the subsection on Model Calibration"*, *"the paragraph following the GP Surrogate subsection"*, *"the merged Methods section"*
- **Wrong:** *"Section 3.2"*, *"line 47"*

Section numbers are generated by LaTeX from document structure. Every time a subsection is added, removed, or merged, all numbers below it shift. Numbers are resolved in a single final pass **after all manuscript edits are complete and the structure is frozen**. Writing numbers during the loop guarantees they will be wrong by the end.

---

## Phase 2 — Iterative response loop

Run up to 5 passes. After pass 3, run the self-check below to decide whether to continue.

### Pass 1 — Address all Reviewer 1 comments

For each R1 comment (skipping Already addressed ones):

1. Read the comment carefully in the context of `main.tex`.
2. Decide: is the reviewer right?
   - **If right:** acknowledge it sincerely. Draft a response that thanks them and describes the change.
   - **If partly right:** acknowledge the valid part, explain the limit of the change.
   - **If wrong or not feasible:** draft a polite, evidence-based rebuttal. Never be dismissive. Use "we respectfully disagree" or "we appreciate this concern; however..." Frame arguments scientifically.
3. If a manuscript change is needed: make the edit in `main.tex` now. Note the section and what changed.
4. If the comment cannot be fully addressed (goes deeper than editing can fix): mark as **Fully unresolved** or **Partly unresolved** with a clear explanation of why.
5. Record in `review_log_R[n].md`.

### Pass 2 — Address all Reviewer 2+ comments

Same process as Pass 1. Additionally:
- Where a Reviewer 2 comment overlaps with a Reviewer 1 comment already addressed, cross-reference: *"This point was also raised by Reviewer 1 (R1.2); please see our response above."*
- Where Reviewer 2 contradicts Reviewer 1, flag it explicitly in `review_log_R[n].md` as a **Reviewer conflict** — do not silently pick one side. Draft a response that acknowledges both perspectives and explains the position taken, or flag for user decision if genuinely irresolvable.

### Pass 3 — Integration and consistency pass

1. Read the full draft `response_R[n].tex` from top to bottom as a coherent document.
2. Check:
   - Does the opening paragraph thank all reviewers warmly and set the right tone?
   - Are cross-cutting comments handled with a unified response (not repeated separately)?
   - Is the tone consistent throughout — polite, constructive, never defensive?
   - Are manuscript change descriptions accurate and specific (section, paragraph, what changed)?
   - Does the letter read as a whole, or does it feel like a patchwork?
3. Fix anything that fails these checks.
4. Re-read `main.tex` to verify all described changes are actually present.

### Self-check after Pass 3

Evaluate honestly:
- Are there any Unaddressed comments remaining? (Should be zero unless flagged as unresolvable)
- Are there any Reviewer conflicts still unresolved?
- Does the letter have any inconsistencies or tonal problems?
- Are all cross-cutting comments handled with a single unified response?

**If all clear:** stop. Proceed to output.
**If issues remain:** run Pass 4 targeting only the specific problems identified.

### Pass 4 (conditional) — Targeted fixes

Address only the specific issues flagged in the self-check. Do not rewrite what is already good.

Run self-check again. If still not clear after Pass 4, run Pass 5.

### Pass 5 (conditional) — Final polish

Final read-through. Tighten language, fix any remaining issues. After Pass 5, stop regardless — report honestly if anything remains unsatisfactory.

## Phase 3 — Structure freeze and number resolution

Run this phase **once**, immediately after the last pass (Pass 3, 4, or 5). All manuscript edits must be complete before this phase — no further changes to `main.tex` after this point.

### Step 1 — Extract section structure

Parse `main.tex` and build a complete map of all `\section`, `\subsection`, and `\subsubsection` commands with their titles and auto-incremented numbers:

```
Section 1 — Introduction
Section 2 — Related Work
Section 3 — Model
  Section 3.1 — Problem Setup
  Section 3.2 — Model Calibration
  Section 3.3 — GP Surrogate
Section 4 — Results
...
```

### Step 2 — Update all references in `response_R[n].tex`

Sweep through `response_R[n].tex` and replace every title-based reference with the full reference (number + title):

- *"the subsection on Model Calibration"* → *"Section 3.2 (Model Calibration)"*
- *"the paragraph following the GP Surrogate subsection"* → *"the paragraph following Section 3.3 (GP Surrogate)"*
- *"the merged Methods section"* → *"Section 3 (Model), which now incorporates..."*

Always keep both number and title in the final reference. If the reviewer compiled a different version and got a slightly different number, the title guides them to the right place.

### Step 3 — Generate `manus_R[n]_diff.tex`

Extract the original manuscript from the snapshot and run `latexdiff`:

```bash
git -C "Overleaf_source" show snapshot-pre-R[n]:main.tex > /tmp/original_R[n].tex
latexdiff /tmp/original_R[n].tex Overleaf_source/main.tex > Overleaf_source/manus_R[n]_diff.tex
rm /tmp/original_R[n].tex
```

This produces a compilable LaTeX file showing all changes highlighted — additions in blue, deletions struck through. Reviewers can see exactly what changed without needing line numbers.

If `latexdiff` is not installed, note it in the final report and skip — do not fail the whole process.

## Output files

### `response_R[n].tex`

Use this LaTeX structure:

```latex
\documentclass[12pt]{article}
\usepackage[margin=2.5cm]{geometry}
\usepackage{xcolor}
\usepackage{parskip}
\usepackage{hyperref}

\newcommand{\comment}[1]{\textbf{Comment:} \textit{#1}}
\newcommand{\response}{\textbf{Response:} }
\newcommand{\change}{\textbf{Manuscript change:} }
\newcommand{\unresolved}[1]{\textcolor{red}{\textbf{[#1]}}}

\begin{document}

\begin{center}
  {\Large \textbf{Response to Reviewers}} \\[6pt]
  {\normalsize <Paper title>} \\[4pt]
  {\small \today}
\end{center}

\bigskip
We thank all reviewers for their careful reading and constructive comments,
which have helped us substantially improve the manuscript.
Below we address each comment in turn.
Changes to the manuscript are indicated in the responses.

%% ─── Cross-cutting comments ──────────────────────────────────────────
\section*{Cross-cutting comments}
%% (only if any exist; otherwise omit this section)

%% ─── Reviewer 1 ──────────────────────────────────────────────────────
\section*{Reviewer 1}

\subsection*{Comment R1.1}
\comment{<verbatim comment text>}

\response <response text>

\change <describe the change and where in the manuscript, or "No manuscript change required.">

%% ... repeat for each comment ...

%% ─── Reviewer 2 ──────────────────────────────────────────────────────
\section*{Reviewer 2}

%% ... same structure ...

%% ─── Unresolved comments ─────────────────────────────────────────────
\section*{Comments requiring further discussion}
%% (only if any exist; otherwise omit)

\subsection*{Comment R1.X — \unresolved{Fully unresolved}}
\comment{<comment text>}

\response <explanation of why this cannot be addressed in the current revision>

\end{document}
```

Use `\unresolved{Partly resolved}` or `\unresolved{Fully unresolved}` tags for flagged comments. These render in red so they are immediately visible.
