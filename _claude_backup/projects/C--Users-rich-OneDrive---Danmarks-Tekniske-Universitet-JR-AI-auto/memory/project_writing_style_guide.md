A writing style guide derived from five reference papers has been set up to govern all text editing and drafting.

**Style guide file:** `C:\Users\rich\OneDrive - Danmarks Tekniske Universitet\JR\AI_auto\literature\Reference_Papers\STYLE_GUIDE.md`

**Reference papers (same folder):**
- Einstein (1905) — On the Electrodynamics of Moving Bodies
- Akerlof (1970) — The Market for Lemons
- Kahneman & Tversky (1979) — Prospect Theory
- Ioannidis (2005) — Why Most Published Research Findings Are False
- Dantzig & Thapa — Linear Programming: Introduction

**How it is wired:** A "Research writing style" section was added to `~/.claude/CLAUDE.md` (global, applies to all projects). It contains the full list of dos and don'ts inline, so agents do not need to load the external file for every edit. The external file has the full explanations.

**Why:** User wants text editing agents to write in the style of these papers rather than generic AI prose. Core principles: economy, concrete before abstract, active voice, no hedging, no filler transitions, one logical load per sentence.
