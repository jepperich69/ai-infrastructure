Overleaf Auto-Sync Infrastructure

Author: Rich Frederiks  |  DTU — Department of Management  |  Updated: March 2026

Purpose: Automatically pull all Overleaf projects to a local folder structure every 4 hours, creating a live local backup that mirrors the Overleaf source — including all subfolders, figures, and bibliography files. Covers publications, projects, teaching material, presentations, and applications.

1. Overview

All Overleaf projects are automatically synchronised to a local folder structure via Overleaf's built-in Git integration. A Windows Scheduled Task triggers the sync every 4 hours. The system covers five categories, each mapped to a different local directory based on the tag assigned in Overleaf:

Overleaf TagLocal DirectoryFolder naming
PapersPublikationer\Manual mapping via papers.csv (Pub_XXX convention)
ProjectsProjekter DTU\Overleaf project name used directly
TeachingKurser\Overleaf project name used directly
PresentationsSlides\Overleaf project name used directly
ApplicationsAnsøgninger\Overleaf project name used directly

Papers use a manual folder mapping (since local Pub_XXX names differ from Overleaf project names). All other categories use the Overleaf project name directly as the folder name — no mapping needed.

3. How It Works

Each Overleaf project exposes a Git remote at https://git.overleaf.com/<project-id>. The sync uses standard git pull — only actual changes are downloaded, so syncing 60+ projects takes only seconds.

Important: the sync is one-way (Overleaf → local). Git will never silently overwrite local files. If a conflict exists, the pull fails and logs ERR — local work is always preserved.

The main script sync_all.ps1 runs in two phases:

  1. Setup phase: reads papers.csv and clones any new papers not yet registered locally.
  2. Sync phase: loops through all entries in projects.json and pulls the latest changes from Overleaf.

5. Adding New Projects

New Paper (tagged "Papers")

1 Create the local folder under Publikationer\, e.g. Pub_NewPaper_TBA\

2 Get the Git URL from Overleaf → Menu → Sync → Git

3 Add a row to papers.csv:

OverleafNameGitUrlLocalFolderSubFolder
My New Paperhttps://git.overleaf.com/XXXXPub_NewPaper_TBAOverleaf_source

4 Done — the next scheduled run clones it automatically.

Alternative (immediate):

.\setup_project.ps1 -FolderName "Pub_NewPaper_TBA" -OverleafUrl "https://git.overleaf.com/XXXX"

New Project / Teaching / Presentation / Application

Simply tag the project correctly in Overleaf (Projects, Teaching, Presentations, or Applications), then run setup_tagged.ps1 once:

.\setup_tagged.ps1

The script reads the tags directly from Overleaf, clones any new projects into the correct local folder using the Overleaf project name, and registers them for the 4-hour sync. No CSV or manual mapping needed.

6. Log Files

8. Requirements

  • Git installed and available on the system PATH
  • Overleaf account with Git sync enabled (available on all paid plans)
  • Git credentials for git.overleaf.com stored in Windows Credential Manager
  • Windows Task Scheduler (built into Windows)
